title: Wombat IDE - Bug fixes and undo/redo date: 2011-09-10T04:55:42 slug: bug-fixes-and-undoredo layout: post categories: - Programming/By Language/Java - Programming/By Language/Scheme - Programming/By Project/Wombat IDE comments: true --- Two minor bug fixes:
  • Fixed the behavior of the bracket matcher so that the functions `<` and `>` actually work.
  • Removed tabs on automatic formatting to avoid problems with different tab widths
And a new feature: Undo/Redo. Now any text field, specifically the REPL and the code editor have an undo/redo stack as they should have had all along. It just uses the Swing Undo/Redo API, so it wasn't terribly hard to implement, but it was something that needed to be done. Hopefully this makes it easier on students that accidently delete things (this may have happened, ergo the new feature :-\).
New version is r178.
