title: PyMint - A Python Multi-Interpreter date: 2007-10-20T04:55:08 slug: pymint-a-python-multi-interpreter layout: post categories: - Programming/By Language/Python comments: true --- During the computer architecture class I took at Rose-Hulman, we were working with a simple assembly language that we had to compile by hand down to @wikipedia["MIPS bytecode"]{MIPS architecture}Â and that's no fun (also there's nothing not worth over doing :)). So I decided to write a program that would allow for modular XML definitions of a language or translation and run it on pretty much any given code. Basically, the XML files have a specific structure to run before, during, and after seeing a specifically structured line in the input (the parser is pretty dump, parsing something like Java would be well beyond it at this point) and run some Python code on that output, transforming it in a number of ways. At the moment, I have definitions that can do any of the following:
  • assemble the given code into bytecode
  • write out those bytes in a structure we can load directly into the simulator
  • even run a virtual machine over the given code.
